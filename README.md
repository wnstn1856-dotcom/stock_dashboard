# 국내·해외 주식 스크리닝 대시보드

PER, PBR, RSI, MACD, 등락률, 거래량, 뉴스 헤드라인(호재/악재) 등 무료 공개 지표를 종합 스코어링해서
국내(KOSPI/KOSDAQ)와 해외(미국) 주식을 한 화면에서 비교해볼 수 있는 로컬 프로그램입니다. 결과는 브라우저에서
여는 HTML 대시보드로 저장됩니다.

## 자동/실시간에 가깝게 쓰고 싶다면

PC를 꺼둬도 30분마다 자동으로 갱신되고 링크 하나로 어디서든 볼 수 있게 하고 싶다면
`GITHUB_자동갱신_설정법.md` 파일을 참고하세요 (GitHub Actions + GitHub Pages, 완전 무료, 마우스 클릭
위주로 설명되어 있습니다).

## 중요: 실시간 시세가 아니고, 자동으로 갱신되지도 않습니다

- 무료 API 특성상 실시간 체결가가 아니라 **지연 시세**(대략 15~20분 지연, 종목/API에 따라 다름)를 사용합니다.
- 이 프로그램은 **실행할 때마다 그 순간의 스냅샷**을 만드는 방식입니다. 저절로 매일 갱신되지 않으며,
  최신 데이터를 보려면 `python main.py`를 다시 실행해야 합니다. 매일 자동으로 돌리고 싶다면 아래
  "매일 자동으로 갱신하고 싶다면" 항목을 참고해 Windows 작업 스케줄러에 등록하세요.
- 진짜 실시간 시세가 필요하면 증권사 Open API(키움, 한국투자증권 등) 연동이 필요하며, 이 프로그램은 그 확장을
  염두에 두고 데이터 수집 부분(`data_kr.py`, `data_us.py`)만 갈아끼우면 되도록 모듈화되어 있습니다.

## 설치

이 프로그램은 인터넷 접속이 되는 **본인 컴퓨터**에서 실행해야 합니다.

```bash
cd stock_dashboard
pip install -r requirements.txt
```

## 실행

```bash
python main.py
```

**종목 수를 크게 늘리고(국내 최대 600, 해외 최대 200) MACD·뉴스 수집까지 추가하면서 실행 시간이 꽤 길어졌습니다
(환경에 따라 20분~1시간 이상 걸릴 수 있습니다).** 종목당 시세 이력 조회 1회 + 국내는 네이버 뉴스 스크래핑
1회가 추가로 붙기 때문입니다. 옵션으로 속도를 조절할 수 있습니다.

```bash
python main.py --no-news     # 뉴스 수집을 건너뛰어 훨씬 빠르게 실행
python main.py --demo        # 네트워크 없이 샘플 데이터로 화면만 확인
```

종목 수 자체를 줄이고 싶으면 `config.py`의 `KR_TOP_N_BY_MARKETCAP`, `US_UNIVERSE_SIZE`를 낮추세요.

- 실행하면 국내 종목(pykrx) → 해외 종목(yfinance) 순으로 시세·펀더멘털·RSI·MACD·뉴스를 수집하고, 종합점수를
  계산한 뒤 `dashboard_YYYYMMDD_HHMM.html` 파일을 생성합니다.
- 생성된 HTML 파일을 더블클릭해서 브라우저로 열면 됩니다. (인터넷 연결 없이도 열람 가능한 단일 파일입니다)
- 장중 데이터가 아직 확정되지 않았거나 휴장일이면, 자동으로 하루씩 이전 영업일로 내려가며 데이터를 찾습니다.
- 국내 뉴스는 네이버 금융 페이지를 스크래핑합니다(비공식 방식). 해외 뉴스는 yfinance가 제공하는 야후
  파이낸스 뉴스를 사용합니다. 둘 다 사이트 구조가 바뀌면 일부 종목에서 실패할 수 있으나, 실패해도 프로그램
  전체는 계속 진행됩니다(해당 종목은 "중립"으로 표시).

## 화면 구성

- **탭**: KOSPI / KOSDAQ / US 시장 전환
- **테이블**: 종목별 현재가, 등락률, PER(업종평균 PER 함께 표시), PBR, RSI, MACD(히스토그램 값 + 골든/데드크로스
  표시), 상대거래량, 뉴스(호재/악재/중립 배지 + 최신 헤드라인), 종합점수, 매수/매도 신호 배지.
  컬럼 헤더 클릭 시 정렬, 업종/테마 필터, 종목 검색, "과매도만"/"매수신호만"/"매도신호만"/"뉴스호재만" 체크박스
  필터 가능
- **종합점수 Top 10 차트**
- **업종/테마별 평균 점수·평균 PER Top 8**

## 스코어링 및 신호 방식 (`scoring.py`)

- **밸류에이션 점수**: 같은 시장 내에서 PER·PBR가 낮을수록 높은 점수 (백분위 기준, 적자/이상치는 낮은 점수 처리)
- **모멘텀 점수**: 등락률과 평소 대비 거래량(상대거래량)이 높을수록 높은 점수
- **종합점수** = 밸류에이션 50% + 모멘텀 50% (가중치는 `config.py`의 `WEIGHT_VALUATION`, `WEIGHT_MOMENTUM`에서 조정)
- **업종평균 PER/PBR**: 같은 시장·같은 업종 내 평균과 비교해 개별 종목이 업종 대비 싼지/비싼지 표시
- **RSI(14일) 기반 과매도/과매수**: RSI가 `OVERSOLD_RSI`(기본 30) 이하면 "과매도", `OVERBOUGHT_RSI`(기본 70)
  이상이면 "과매수"
- **MACD(12,26,9) 골든크로스/데드크로스**: MACD 히스토그램이 음수→양수로 전환되면 "골든크로스"(매수 참고),
  양수→음수로 전환되면 "데드크로스"(매도 참고)
- **뉴스 호재/악재**: 최신 헤드라인 몇 개를 단순 키워드 매칭으로 스캔해 "호재"/"악재"/"중립" 태그 부여
  (진짜 AI 감성분석이 아닌 규칙 기반 참고 지표)
- **매수 신호(참고용)**: 과매도·저평가·거래량급증·MACD골든크로스·뉴스호재 5개 중 `BUY_SIGNAL_MIN_COUNT`
  (기본 3개) 이상 겹치면 "매수신호" 배지
- **매도 신호(참고용)**: 과매수·MACD데드크로스·뉴스악재 3개 중 `SELL_SIGNAL_MIN_COUNT`(기본 2개) 이상 겹치면
  "매도신호" 배지

**이 신호들은 모두 규칙을 기계적으로 조합한 참고 지표일 뿐, 투자 자문이나 실제 매수·매도 추천이 아닙니다.**

## 종목/신호 기준 조정 (`config.py`)

- `KR_MARKETS`, `KR_TOP_N_BY_MARKETCAP`(기본 300, 시장당): 국내 대상 시장과 시가총액 상위 몇 종목까지 볼지
- `US_UNIVERSE_SIZE`(기본 200), `US_FALLBACK_TICKERS`: 해외는 위키피디아 S&P500 목록을 우선 시도하고, 실패 시
  curated 리스트 사용
- `RSI_PERIOD`, `MACD_FAST/SLOW/SIGNAL`, `KR_HISTORY_DAYS`, `US_HISTORY_PERIOD`: 기술적 지표 계산 기간
- `FETCH_NEWS`, `NEWS_MAX_ITEMS`, `NEWS_SLEEP_SEC`: 뉴스 수집 여부/개수/요청 간 대기시간
- `OVERSOLD_RSI`, `OVERBOUGHT_RSI`, `VOLUME_SURGE_MULT`, `UNDERVALUED_PERCENTILE`, `BUY_SIGNAL_MIN_COUNT`,
  `SELL_SIGNAL_MIN_COUNT`: 신호 판단 기준값

## 매일 자동으로 갱신하고 싶다면 (Windows 작업 스케줄러)

이 프로그램 자체는 정적 파일을 생성하는 방식이라 실행할 때마다 새 스냅샷을 만듭니다. 매일 아침 자동으로
새 대시보드를 만들고 싶다면 Windows 작업 스케줄러에 등록하세요. 단, 스케줄러는 컴퓨터가 켜져 있고
인터넷에 연결된 상태에서만 동작하며, 실행 시간이 길어졌으니(최대 1시간 내외) 트리거 시간을 여유 있게 잡으세요.

1. 시작 메뉴에서 "작업 스케줄러" 검색 후 실행
2. 오른쪽 "작업 만들기" 클릭
3. 일반 탭: 이름 입력 (예: 주식 대시보드 갱신)
4. 트리거 탭: "새로 만들기" → 매일, 원하는 시간(예: 오전 7시) 설정
5. 동작 탭: "새로 만들기" →
   - 프로그램/스크립트: `python` (또는 `python.exe`의 전체 경로)
   - 인수 추가: `main.py` (빠르게 돌리고 싶으면 `main.py --no-news`)
   - 시작 위치: `stock_dashboard` 폴더의 전체 경로 (예: `C:\Users\ibr\Desktop\클로드\stock_dashboard`)
6. 확인 후 저장

이렇게 설정하면 매일 지정한 시간에 새 `dashboard_YYYYMMDD_HHMM.html` 파일이 폴더에 쌓입니다. (오래된 파일은
수동으로 정리해야 합니다.)

## 한계 및 주의사항

- 업종/테마 분류는 KRX 상장법인 업종 코드 및 야후파이낸스 GICS 섹터를 사용한 근사치이며, 실제 증권사 앱의
  "테마주" 분류와는 다를 수 있습니다.
- 뉴스 호재/악재 판정은 헤드라인에 특정 키워드가 있는지만 보는 단순 규칙이며, 문맥을 이해하지 못합니다
  (예: "적자 축소"도 "적자"라는 단어 때문에 악재로 오분류될 수 있음). 참고용으로만 사용하세요.
- PER이 낮다고, RSI가 낮다고(과매도), MACD가 골든크로스라고 반드시 좋은 투자처는 아닙니다. "매수 신호"/"매도
  신호" 배지는 여러 규칙을 기계적으로 조합한 것일 뿐이며 실제 주가 방향을 보장하지 않습니다.
- 이 대시보드는 1차 스크리닝 도구이며, 최종 투자 판단은 반드시 재무제표, 공시, 원문 뉴스를 직접 확인한 뒤
  본인 책임하에 내려야 합니다.
- 무료 API/스크래핑은 요청량이 많으면 일시적으로 차단되거나 느려질 수 있습니다. 특히 국내 뉴스 수집은
  종목당 네이버에 요청을 1건씩 보내므로 종목 수를 크게 늘리면 시간이 비례해서 늘어납니다.
